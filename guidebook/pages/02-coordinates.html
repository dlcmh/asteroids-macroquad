Chapter 2

Screen Coordinates & Drawing

The Coordinate System

Computer screens use a coordinate system that might surprise you:

  • X goes right (like math class)
  • Y goes DOWN (opposite of math class!)
  • The origin (0,0) is the TOP-LEFT corner
(0,0) ──────────────► X increases (right)
  │
  │    Screen
  │
  ▼
Y increases (down)

This is why we use screen_height - y sometimes when we want "math-style" coordinates.

Getting Screen Dimensions

macroquad gives us the window size dynamically:

let screen_w = screen_width();
let screen_h = screen_height();

// Center of the screen
let center_x = screen_w / 2.0;
let center_y = screen_h / 2.0;

The center is where your ship will start. Remember: f32 not i32 — everything in graphics uses floating point!

Drawing Shapes

macroquad gives us simple drawing functions. Here are the ones we'll use most:

Lines

draw_line(x1, y1, x2, y2, thickness, color);

Draws a line from (x1,y1) to (x2,y2). Thickness in pixels.

Circles

draw_circle(x, y, radius, color);

Center at (x,y) with given radius.

Triangles (for ships!)

draw_triangle(
    Vec2::new(x1, y1),
    Vec2::new(x2, y2),
    Vec2::new(x3, y3),
    color
);

Three points define a triangle. Vec2 is just an (x,y) pair.

Your First Drawing: A Box at Center

use macroquad::prelude::*;

#[macroquad::main("Coordinates")]
async fn main() {
    loop {
        clear_background(BLACK);
        
        let cx = screen_width() / 2.0;
        let cy = screen_height() / 2.0;
        
        // Draw a crosshair at center
        draw_line(cx - 20.0, cy, cx + 20.0, cy, 2.0, WHITE);
        draw_line(cx, cy - 20.0, cx, cy + 20.0, 2.0, WHITE);
        draw_circle(cx, cy, 5.0, GREEN);
        
        // Draw our "ship" for now - just a circle
        draw_circle(cx, cy - 50.0, 15.0, YELLOW);
        
        next_frame().await;
    }
}

Run this. You'll see a crosshair at center and a yellow circle above it. That circle will become our ship.

The Vec2 Type

You'll see Vec2 everywhere. It's just:

struct Vec2 {
    x: f32,
    y: f32,
}

But it comes with useful math operations:

let position = Vec2::new(100.0, 200.0);
let velocity = Vec2::new(1.5, -0.5);

// Move position by velocity
let new_pos = position + velocity;  // (101.5, 199.5)

// Scale velocity
let fast = velocity * 2.0;  // (3.0, -1.0)

We'll use Vec2 for positions, velocities, and directions throughout this project.
