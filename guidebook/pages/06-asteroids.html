Chapter 6

Asteroids!

Irregular rocks, random motion, and the screen-wrapping menace that makes this game worth playing.

What Makes an Asteroid?

An asteroid is just a lumpy circle. We fake the lumpiness by generating a handful of random points around a center, then connecting them into a polygon. Each asteroid gets its own unique shape, so no two look exactly alike.

struct Asteroid {
    position: Vec2,       // Where it is
    velocity: Vec2,       // How it drifts
    rotation: f32,        // Current spin angle
    rotation_speed: f32,  // How fast it spins
    radius: f32,          // Approximate size (for collisions later)
    shape: Vec<Vec2>,     // The actual polygon vertices
}

shape — A list of points relative to the asteroid's center. Generated once when the asteroid spawns. These are offsets, not screen positions—we add position to them when drawing.

rotation_speed — Each asteroid spins at a different rate. Some tumble slowly, others whirl. All cosmetic, but it sells the "rocks in space" feel.

radius — We'll need this in Chapter 8 for collision detection. It's the average distance from center to the shape's vertices. A rough circle approximation.

Generating Irregular Shapes

Here's the trick: we pick a fixed number of points (say 10), space them evenly around a circle, then randomly jitter their distance from the center. Connect them in order and you get a believable asteroid.

fn generate_asteroid_shape(radius: f32) -> Vec<Vec2> {
    let num_vertices = 10;
    let mut vertices = Vec::with_capacity(num_vertices);

    for i in 0..num_vertices {
        // Evenly spaced angle around the circle
        let angle = (i as f32 / num_vertices as f32) * 2.0 * std::f32::consts::PI;

        // Random distance: between 70% and 100% of the radius
        let jitter = rand::gen_range(0.7, 1.0);
        let dist = radius * jitter;

        vertices.push(Vec2::new(
            angle.cos() * dist,
            angle.sin() * dist,
        ));
    }

    vertices
}

🧠 Why 70% to 100%?

If vertices could go all the way to zero, you'd get needle-thin spikes pointing at the center—unrealistic for a rock. Keeping them within 70–100% of the radius ensures the asteroid stays chunky and believable. If you want spikier asteroids, change the range to 0.4–1.0. Smoother? Try 0.85–1.0.

Drawing a Rotated Polygon

Each frame, we need to take the stored shape vertices, rotate them by the asteroid's current spin angle, offset them to the asteroid's screen position, and connect them with lines.

fn draw_asteroid(asteroid: &Asteroid) {
    let cos = asteroid.rotation.cos();
    let sin = asteroid.rotation.sin();

    // Rotate and offset each vertex
    let world_vertices: Vec<Vec2> = asteroid.shape.iter().map(|v| {
        // Rotate the vertex by the current spin angle
        let rx = v.x * cos - v.y * sin;
        let ry = v.x * sin + v.y * cos;
        // Offset to world position
        Vec2::new(
            asteroid.position.x + rx,
            asteroid.position.y + ry,
        )
    }).collect();

    // Draw lines between consecutive vertices
    for i in 0..world_vertices.len() {
        let start = world_vertices[i];
        let end = world_vertices[(i + 1) % world_vertices.len()];
        draw_line(start.x, start.y, end.x, end.y, 2.0, WHITE);
    }
}

🧠 Rotation Math Refresher

To rotate a point (x, y) by angle θ:

new_x = x * cos(θ) - y * sin(θ)
new_y = x * sin(θ) + y * cos(θ)

This is different from the ship drawing in Chapter 5, where we computed rotated points directly using cos/sin of the angle. Here we're rotating pre-stored vertices. Both approaches are valid—this one is cleaner when you have a list of points to rotate together.

💡 The (i + 1) % len() Trick

We need to draw a line from vertex 0 to 1, 1 to 2, ..., and finally the last vertex back to vertex 0. The modulo operator handles that wraparound: when i is the last index, (i + 1) % len gives 0. Clean and classic.

Spawning Asteroids

We need asteroids to appear at random positions along the edges of the screen, drifting inward at random speeds. Here's the spawn function:

fn spawn_asteroid(screen_w: f32, screen_h: f32) -> Asteroid {
    // Pick a random edge: 0=top, 1=right, 2=bottom, 3=left
    let edge = rand::gen_range(0, 4);

    let position = match edge {
        0 => Vec2::new(rand::gen_range(0.0, screen_w), 0.0),              // Top
        1 => Vec2::new(screen_w, rand::gen_range(0.0, screen_h)),         // Right
        2 => Vec2::new(rand::gen_range(0.0, screen_w), screen_h),         // Bottom
        _ => Vec2::new(0.0, rand::gen_range(0.0, screen_h)),              // Left
    };

    // Random velocity: drifting generally inward, not too fast
    let angle = rand::gen_range(0.0, 2.0 * std::f32::consts::PI);
    let speed = rand::gen_range(30.0, 80.0);
    let velocity = Vec2::new(angle.cos() * speed, angle.sin() * speed);

    // Random size: big, medium, or small
    let radius = rand::gen_range(20.0, 50.0);

    Asteroid {
        position,
        velocity,
        rotation: rand::gen_range(0.0, 2.0 * std::f32::consts::PI),
        rotation_speed: rand::gen_range(-1.5, 1.5),
        radius,
        shape: generate_asteroid_shape(radius),
    }
}

🚨 The Edge Spawning Problem

Asteroids that spawn right on the edge might immediately wrap to the other side on the first frame. That's actually fine in Asteroids—the wrapping is part of the game. But if it bothers you, spawn them slightly outside the screen bounds (e.g., -radius or screen_w + radius) so they drift in naturally.

The Complete Asteroid Module

Everything in one place. Save this as asteroid.rs:

use macroquad::prelude::*;

pub struct Asteroid {
    pub position: Vec2,
    pub velocity: Vec2,
    pub rotation: f32,
    pub rotation_speed: f32,
    pub radius: f32,
    pub shape: Vec<Vec2>,
}

impl Asteroid {
    pub fn new(screen_w: f32, screen_h: f32, radius: f32) -> Self {
        let edge = rand::gen_range(0, 4);
        let position = match edge {
            0 => Vec2::new(rand::gen_range(0.0, screen_w), 0.0),
            1 => Vec2::new(screen_w, rand::gen_range(0.0, screen_h)),
            2 => Vec2::new(rand::gen_range(0.0, screen_w), screen_h),
            _ => Vec2::new(0.0, rand::gen_range(0.0, screen_h)),
        };

        let angle = rand::gen_range(0.0, 2.0 * std::f32::consts::PI);
        let speed = rand::gen_range(30.0, 80.0);

        Self {
            position,
            velocity: Vec2::new(angle.cos() * speed, angle.sin() * speed),
            rotation: rand::gen_range(0.0, 2.0 * std::f32::consts::PI),
            rotation_speed: rand::gen_range(-1.5, 1.5),
            radius,
            shape: generate_shape(radius),
        }
    }

    pub fn update(&mut self, dt: f32) {
        self.position += self.velocity * dt;
        self.rotation += self.rotation_speed * dt;
        wrap_position(&mut self.position);
    }

    pub fn draw(&self) {
        let cos = self.rotation.cos();
        let sin = self.rotation.sin();

        let world_vertices: Vec<Vec2> = self.shape.iter().map(|v| {
            let rx = v.x * cos - v.y * sin;
            let ry = v.x * sin + v.y * cos;
            Vec2::new(self.position.x + rx, self.position.y + ry)
        }).collect();

        for i in 0..world_vertices.len() {
            let start = world_vertices[i];
            let end = world_vertices[(i + 1) % world_vertices.len()];
            draw_line(start.x, start.y, end.x, end.y, 2.0, WHITE);
        }
    }
}

fn generate_shape(radius: f32) -> Vec<Vec2> {
    let num_vertices = 10;
    let mut vertices = Vec::with_capacity(num_vertices);

    for i in 0..num_vertices {
        let angle = (i as f32 / num_vertices as f32) * 2.0 * std::f32::consts::PI;
        let jitter = rand::gen_range(0.7, 1.0);
        let dist = radius * jitter;
        vertices.push(Vec2::new(angle.cos() * dist, angle.sin() * dist));
    }

    vertices
}

fn wrap_position(pos: &mut Vec2) {
    let w = screen_width();
    let h = screen_height();
    if pos.x < 0.0 { pos.x += w; }
    if pos.x > w    { pos.x -= w; }
    if pos.y < 0.0 { pos.y += h; }
    if pos.y > h    { pos.y -= h; }
}

Updated main.rs with Asteroids

Now we bring it all together. Your game loop spawns a few asteroids and updates them alongside the ship:

use macroquad::prelude::*;

mod ship;
mod asteroid;
use ship::Ship;
use asteroid::Asteroid;

#[macroquad::main("Asteroids")]
async fn main() {
    let mut ship = Ship::new();

    // Start with 4 asteroids
    let mut asteroids: Vec<Asteroid> = (0..4)
        .map(|_| Asteroid::new(screen_width(), screen_height(), 40.0))
        .collect();

    loop {
        let dt = get_frame_time();
        clear_background(BLACK);

        // Input
        let input = ship::get_input();

        // Update ship
        ship.update(&input, dt);

        // Update asteroids
        for asteroid in &mut asteroids {
            asteroid.update(dt);
        }

        // Draw asteroids
        for asteroid in &asteroids {
            asteroid.draw();
        }

        // Draw ship (on top)
        ship.draw();

        next_frame().await;
    }
}

Run it. You should see your ship at center with four lumpy polygons drifting and tumbling across the screen. They'll wrap around the edges. You can fly through them harmlessly—for now.

Asteroid Sizes: The Splitting Mechanic

In the real Asteroids game, shooting a large asteroid splits it into two medium ones. Shooting a medium splits it into two small ones. Shooting a small destroys it. Each size has its own radius and point value. We'll implement the full splitting logic in Chapter 8 (collisions), but here's the size system we'll use:

Size Radius Points Splits Into
Large 40–50 20 2 Medium
Medium 25–35 50 2 Small
Small 12–20 100 Destroyed

Smaller asteroids are worth more points because they're harder to hit. We'll add a size field to the Asteroid struct in Chapter 8.

Key Takeaways

Jittered Vertices: Random distance, fixed angle

The simplest way to generate organic-looking shapes. Ten vertices is plenty.

Edge Spawning: Pick a random side of the screen

Ensures asteroids enter from all directions. Combined with wrapping, they never leave.

Rotation Math: Rotate vertices, then offset by position

The (x*cos - y*sin, x*sin + y*cos) formula is worth memorizing. You'll use it everywhere.

Vec of Asteroids: Dynamic list, iterated each frame

We'll add and remove asteroids as they split and get destroyed. Vectors handle this naturally.
