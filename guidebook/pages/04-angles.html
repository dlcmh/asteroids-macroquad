Chapter 4

Angles, Radians & atan2

The three things that scare beginners the most. They're simpler than you think.

Why Radians Instead of Degrees?

You grew up with degrees. A circle has 360°. It's arbitrary—some ancient Babylonian decided this thousands of years ago because 360 divides nicely by lots of numbers.

Radians are the universe's native angle unit. One radian is the angle you get when you wrap the radius of a circle along its circumference.

Full circle = 2π radians ≈ 6.283

That's it. That's the only fact you need to memorize.

Degrees Radians What it means for your ship
0° 0 Pointing right →
90° π/2 ≈ 1.57 Pointing down ↓
180° π ≈ 3.14 Pointing left ←
270° 3π/2 ≈ 4.71 Pointing up ↑
360° 2π ≈ 6.28 Back to pointing right →

⚠️ Wait—0° points RIGHT, not UP?

Yes. In math and game programming, angle 0 is the positive X axis (right), and angles increase counterclockwise. This is the standard convention. Get comfortable with it now—every math function you'll use assumes this.

macroquad gives you π as std::f32::consts::PI. Use it. Never type 3.14—it's not precise enough.

Converting an Angle to a Direction Vector

Your ship has an angle. You press "thrust." Which way is forward? You need to convert the angle into X and Y components.

direction_x = cos(angle)

direction_y = sin(angle)

Let's test this with the table above:

// Angle 0 (pointing right)
cos(0) = 1.0   // Full X, no Y → right
sin(0) = 0.0

// Angle PI/2 (pointing down)
cos(PI/2) = 0.0   // No X, full Y → down
sin(PI/2) = 1.0

// Angle PI (pointing left)
cos(PI) = -1.0   // Negative X, no Y → left
sin(PI) = 0.0

// Angle 3*PI/2 (pointing up)
cos(3*PI/2) = 0.0   // No X, negative Y → up
sin(3*PI/2) = -1.0

It works! Cosine gives you the X component, sine gives you the Y component. Always. For any angle.

let forward_x = ship.angle.cos();
let forward_y = ship.angle.sin();
let forward = Vec2::new(forward_x, forward_y);

// Apply thrust in the forward direction
ship.velocity += forward * thrust_power * dt;

🧠 Mental Model

Imagine a circle of radius 1 centered at the origin. For any point on that circle, cos(angle) is the X coordinate, sin(angle) is the Y coordinate. That's literally the definition of sine and cosine. When you thrust, you're just moving in the direction of that point on the circle.

atan2: The Function You'll Actually Need

We just went from angle to direction. But sometimes you have a direction and need the angle. Specifically: you know where the mouse is, and you want your ship to point at it.

angle = atan2(y, x)

Note: Y comes FIRST. This catches everyone.

atan2 takes a Y and an X, and returns the angle from the origin to that point. It handles all the quadrant math for you.

// Where is the mouse relative to the ship?
let dx = mouse_x - ship.position.x;
let dy = mouse_y - ship.position.y;

// What angle points from ship to mouse?
let target_angle = dy.atan2(dx);
// YES: dy.atan2(dx), NOT dx.atan2(dy)

🚨 Why atan2 and not atan?

atan(y/x) can't tell the difference between (1,1) and (-1,-1) because the division loses sign information. atan2(y, x) takes both separately and returns the correct angle for all four quadrants. Always use atan2. Never atan.

Rotating the Ship

To turn the ship, you just add or subtract from its angle:

let rotation_speed = 3.0; // Radians per second

if is_key_down(KeyCode::Left) {
    ship.angle -= rotation_speed * dt;
}
if is_key_down(KeyCode::Right) {
    ship.angle += rotation_speed * dt;
}

But there's a problem: after many rotations, your angle could be 100π. That's fine mathematically, but for display and debugging, it's cleaner to keep angles in [0, 2π):

// Normalize angle to [0, 2π)
fn normalize_angle(angle: f32) -> f32 {
    let mut a = angle % (2.0 * std::f32::consts::PI);
    if a < 0.0 {
        a += 2.0 * std::f32::consts::PI;
    }
    a
}

Not strictly required, but good practice. It prevents floating-point drift over long play sessions.

Drawing a Rotated Ship

Now the payoff. Instead of a circle, let's draw a triangle that actually points in the ship's direction:

fn draw_ship(position: Vec2, angle: f32) {
    // Ship points right by default (angle 0 = right)
    let nose = Vec2::new(
        position.x + angle.cos() * 20.0,
        position.y + angle.sin() * 20.0,
    );

    // Back-left corner: angle + 150° (in radians: 5π/6)
    let left_wing = Vec2::new(
        position.x + (angle + 5.0 * std::f32::consts::PI / 6.0).cos() * 15.0,
        position.y + (angle + 5.0 * std::f32::consts::PI / 6.0).sin() * 15.0,
    );

    // Back-right corner: angle - 150°
    let right_wing = Vec2::new(
        position.x + (angle - 5.0 * std::f32::consts::PI / 6.0).cos() * 15.0,
        position.y + (angle - 5.0 * std::f32::consts::PI / 6.0).sin() * 15.0,
    );

    draw_triangle(nose, left_wing, right_wing, WHITE);
}

🧠 What's happening here?

We're computing the three corners of the triangle by starting from the ship's center position and moving outward in three directions. The nose points in the ship's angle. The wings are offset by ±150° from that direction. The 20.0 and 15.0 are distances—the nose is longer than the wings are wide.

As angle changes (from pressing Left/Right), all three points rotate together. The triangle always points the right way.

Complete Rotating Ship Example

Replace your entire main.rs with this:

use macroquad::prelude::*;

struct Ship {
    position: Vec2,
    velocity: Vec2,
    angle: f32,
}

fn wrap_position(pos: &mut Vec2, screen_w: f32, screen_h: f32) {
    if pos.x < 0.0 { pos.x += screen_w; }
    if pos.x > screen_w { pos.x -= screen_w; }
    if pos.y < 0.0 { pos.y += screen_h; }
    if pos.y > screen_h { pos.y -= screen_h; }
}

fn draw_ship(position: Vec2, angle: f32) {
    let nose = Vec2::new(
        position.x + angle.cos() * 20.0,
        position.y + angle.sin() * 20.0,
    );
    let left_wing = Vec2::new(
        position.x + (angle + 2.618).cos() * 15.0,
        position.y + (angle + 2.618).sin() * 15.0,
    );
    let right_wing = Vec2::new(
        position.x + (angle - 2.618).cos() * 15.0,
        position.y + (angle - 2.618).sin() * 15.0,
    );
    draw_triangle(nose, left_wing, right_wing, WHITE);
}

#[macroquad::main("Rotating Ship")]
async fn main() {
    let mut ship = Ship {
        position: Vec2::new(screen_width() / 2.0, screen_height() / 2.0),
        velocity: Vec2::new(0.0, 0.0),
        angle: 0.0,
    };

    loop {
        let dt = get_frame_time();
        clear_background(BLACK);

        // Rotation
        let rotation_speed = 4.0;
        if is_key_down(KeyCode::Left)  { ship.angle -= rotation_speed * dt; }
        if is_key_down(KeyCode::Right) { ship.angle += rotation_speed * dt; }

        // Thrust
        let thrust_power = 200.0;
        if is_key_down(KeyCode::Up) {
            ship.velocity.x += ship.angle.cos() * thrust_power * dt;
            ship.velocity.y += ship.angle.sin() * thrust_power * dt;
        }

        // Update position
        ship.position += ship.velocity * dt;
        wrap_position(&mut ship.position, screen_width(), screen_height());

        // Slight drag
        ship.velocity *= 0.99;

        // Draw
        draw_ship(ship.position, ship.angle);

        next_frame().await;
    }
}

Run it. Arrow keys turn and thrust. The ship drifts with momentum. You're flying!

Note: I used 2.618 instead of 5.0 * PI / 6.0 for brevity. In your actual code, use the full constant for clarity.

Key Takeaways

Radians: 2π = 360°. Use std::f32::consts::PI.

Let go of degrees. Radians are what the math functions expect.

Angle → Direction: (cos(θ), sin(θ))

This pair always gives a unit vector in the direction of θ.

Direction → Angle: atan2(y, x)

Y first. Always Y first. You will forget this at least once.

Rotating Points: Add offset to angle before cos/sin

The ship's wings are just the nose angle ± some offset. Clean and simple.
