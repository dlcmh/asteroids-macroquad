Chapter 1

Getting Started

What We're Building

Asteroids is the perfect first game. It has everything you need to learn:

  • Player input handling
  • Movement and physics simulation
  • Collision detection
  • Object spawning and destruction
  • Score tracking and game state

And it's simple enough that we can build it piece by piece, understanding every line of code.

Project Setup

cargo new asteroids-macroquad

Then add the macroquad dependency:

cargo add macroquad

Your First Game Loop

Every game has a "game loop" that runs 60 times per second. It does three things:

  1. Process input - What keys are pressed?
  2. Update state - Move things, check collisions
  3. Draw - Render everything to the screen
use macroquad::prelude::*;

#[macroquad::main("Asteroids")]
async fn main() {
    loop {
        // 1. Process input
        // 2. Update state  
        // 3. Draw everything
        
        clear_background(BLACK);
        draw_text("Hello, Asteroids!", 20.0, 20.0, 30.0, WHITE);
        
        next_frame().await;
    }
}

Run it with cargo run. You'll see a black window with white text. That's your canvas!

Key Concepts

clear_background(BLACK)

Erases the previous frame. Without this, things "smear" across the screen.

next_frame().await

Tells macroquad "I'm done drawing, show it on screen, then wait for the next frame." This is what gives you 60 FPS.

async fn main()

macroquad uses async Rust. Don't worry about this yet—just copy the pattern. The #[macroquad::main] attribute handles the setup for you.

Checking Your Work

✅ Compiles without errors
✅ Opens a window
✅ Shows "Hello, Asteroids!" text
✅ Closes cleanly when you close the window

If all four check out, you're ready for Chapter 2!
