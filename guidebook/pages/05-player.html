Chapter 5

The Player Ship

Turning a triangle into a spaceship. Thrust flames, responsive controls, and the first real "feel" of the game.

Designing the Ship Shape

The classic Asteroids ship is a simple triangle—pointed at the nose, wide at the back. We already drew one in Chapter 4, but let's refine the proportions and understand what makes it look like a ship.

// Ship dimensions (tweak these to your taste)
const SHIP_LENGTH: f32 = 20.0;  // Nose to center
const SHIP_WIDTH: f32 = 12.0;   // Half the base width
const WING_ANGLE: f32 = 2.618;  // 150° in radians (5π/6)

fn draw_ship(position: Vec2, angle: f32) {
    // Nose: directly ahead
    let nose = Vec2::new(
        position.x + angle.cos() * SHIP_LENGTH,
        position.y + angle.sin() * SHIP_LENGTH,
    );

    // Left wing: angle - WING_ANGLE — - is counter-clockwise
    let left_wing = Vec2::new(
        position.x + (angle - WING_ANGLE).cos() * SHIP_WIDTH,
        position.y + (angle - WING_ANGLE).sin() * SHIP_WIDTH,
    );

    // Right wing: angle + WING_ANGLE — + is clockwise (Y down)
    let right_wing = Vec2::new(
        position.x + (angle + WING_ANGLE).cos() * SHIP_WIDTH,
        position.y + (angle + WING_ANGLE).sin() * SHIP_WIDTH,
    );

    draw_triangle(nose, left_wing, right_wing, WHITE);
}

🧠 Why 150° for the wings?

Think of the ship as an arrowhead. The nose points forward (0° offset). If the wings were at 180° they'd form a straight line. At 90° they'd make a right angle. At 150° you get a sleek, recognizable ship shape. Try 140° for a wider ship or 160° for a narrower one—it's an artistic choice, not a mathematical one.

The Complete Ship Struct

Our ship needs to track more than just position and velocity now. Here's everything a proper ship needs:

struct Ship {
    position: Vec2,      // Where on screen
    velocity: Vec2,      // Drift speed and direction
    angle: f32,          // Which way it's pointing (radians)
    is_thrusting: bool,  // Is the thruster firing?
    is_dead: bool,       // Did we hit an asteroid?
    invincible_timer: f32, // Brief invincibility after respawn
}

is_thrusting — We need this to know whether to draw the flame. The player might be pressing Up, or not.

is_dead — When the ship hits an asteroid, we don't immediately delete it. We play a death animation, then respawn. This flag tells the game loop "don't let the player fly a dead ship."

invincible_timer — After respawning, the player gets a couple seconds of invulnerability. This timer counts down. While > 0, collisions are ignored. We'll implement this in Chapter 9.

Handling Input Cleanly

Instead of scattering is_key_down calls through the update logic, collect all input at the top of the loop. This makes the ship's behavior easy to read and modify:

struct ShipInput {
    rotate_left: bool,
    rotate_right: bool,
    thrust: bool,
    shoot: bool,
}

fn get_ship_input() -> ShipInput {
    ShipInput {
        rotate_left: is_key_down(KeyCode::Left),
        rotate_right: is_key_down(KeyCode::Right),
        thrust: is_key_down(KeyCode::Up),
        shoot: is_key_pressed(KeyCode::Space), // Single press, not hold
    }
}

💡 is_key_down vs is_key_pressed

is_key_down returns true every frame while the key is held. Use it for continuous actions: turning, thrusting. is_key_pressed returns true only on the frame the key first goes down. Use it for one-shot actions: shooting a bullet, pausing the game.

The Ship Update Function

All the physics from Chapters 3 and 4, now organized into a clean function:

const ROTATION_SPEED: f32 = 4.0;    // Radians per second
const THRUST_POWER: f32 = 300.0;     // Acceleration in pixels/sec²
const DRAG: f32 = 0.98;              // Velocity retained per frame
const MAX_SPEED: f32 = 400.0;        // Speed cap

fn update_ship(ship: &mut Ship, input: &ShipInput, dt: f32) {
    // Rotation
    if input.rotate_left {
        ship.angle -= ROTATION_SPEED * dt;
    }
    if input.rotate_right {
        ship.angle += ROTATION_SPEED * dt;
    }

    // Thrust
    ship.is_thrusting = input.thrust;
    if ship.is_thrusting {
        ship.velocity.x += ship.angle.cos() * THRUST_POWER * dt;
        ship.velocity.y += ship.angle.sin() * THRUST_POWER * dt;
    }

    // Apply drag
    ship.velocity *= DRAG;

    // Clamp speed
    let speed = ship.velocity.length();
    if speed > MAX_SPEED {
        ship.velocity = ship.velocity.normalize_or_zero() * MAX_SPEED;
    }

    // Update position
    ship.position += ship.velocity * dt;
    wrap_position(&mut ship.position, screen_width(), screen_height());
}

🧠 normalize() and normalize_or_zero()

Every vector has a direction and a length. Normalizing keeps the direction but sets the length to exactly 1.0—a "unit vector" pointing the same way.

let v = Vec2::new(300.0, 400.0);
v.length();        // 500.0 (Pythagoras: √(300² + 400²))
v.normalize();     // Vec2(0.6, 0.8) — same direction, length 1.0

Once you have a unit vector, multiply by any speed to get exactly that speed in the original direction. That's what the speed cap does: same heading, controlled magnitude.

normalize_or_zero() is the safe version. A zero-length vector has no direction—normalizing it would divide by zero and crash. normalize_or_zero() just returns Vec2::ZERO instead. Use it whenever the vector might be zero (like a stationary ship).

🚨 Speed Clamping is Important

Without a maximum speed, the player could thrust forever and reach ridiculous velocities. The ship would zip across the screen in a single frame, making the game unplayable. clamp_length or the normalize-and-multiply trick above caps the top speed.

Drawing the Thrust Flame

When the ship thrusts, we draw a flickering flame behind it. The flame is just a triangle that changes size randomly each frame:

fn draw_thrust_flame(position: Vec2, angle: f32) {
    // Random flicker: flame length varies each frame
    let flame_length = rand::gen_range(8.0, 20.0);
    let flame_width = rand::gen_range(3.0, 7.0);

    // The flame comes out the BACK of the ship
    let back_angle = angle + std::f32::consts::PI; // Opposite direction

    // Base of flame (at the ship's back)
    let base = Vec2::new(
        position.x + back_angle.cos() * 8.0,
        position.y + back_angle.sin() * 8.0,
    );

    // Tip of flame (further back, varies with flicker)
    let tip = Vec2::new(
        base.x + back_angle.cos() * flame_length,
        base.y + back_angle.sin() * flame_length,
    );

    // Two sides of the flame triangle
    let left = Vec2::new(
        base.x + (back_angle + 1.5).cos() * flame_width,
        base.y + (back_angle + 1.5).sin() * flame_width,
    );
    let right = Vec2::new(
        base.x + (back_angle - 1.5).cos() * flame_width,
        base.y + (back_angle - 1.5).sin() * flame_width,
    );

    // Fire colors: orange core, red edges
    draw_triangle(tip, left, right, ORANGE);
    draw_triangle(tip, base, right, RED);
}

The key insight: the flame points opposite to the ship's angle. When the ship faces right (angle 0), the flame goes left (angle π). We compute this with angle + PI.

🧠 Why two triangles?

A single triangle looks flat. Two overlapping triangles with slightly different colors create a more natural flame shape. The orange one is tall and narrow (the core), the red one is shorter and wider (the outer glow). It's a cheap trick that looks great.

The Complete Ship Module

Everything we've built, in one place. This is a complete ship.rs module:

use macroquad::prelude::*;

pub struct Ship {
    pub position: Vec2,
    pub velocity: Vec2,
    pub angle: f32,
    pub is_thrusting: bool,
    pub is_dead: bool,
    pub invincible_timer: f32,
}

pub struct ShipInput {
    pub rotate_left: bool,
    pub rotate_right: bool,
    pub thrust: bool,
    pub shoot: bool,
}

const ROTATION_SPEED: f32 = 4.0;
const THRUST_POWER: f32 = 300.0;
const DRAG: f32 = 0.98;
const MAX_SPEED: f32 = 400.0;

impl Ship {
    pub fn new() -> Self {
        Self {
            position: Vec2::new(screen_width() / 2.0, screen_height() / 2.0),
            velocity: Vec2::ZERO,
            angle: 0.0,
            is_thrusting: false,
            is_dead: false,
            invincible_timer: 0.0,
        }
    }

    pub fn update(&mut self, input: &ShipInput, dt: f32) {
        if self.is_dead {
            return;
        }

        // Count down invincibility
        if self.invincible_timer > 0.0 {
            self.invincible_timer -= dt;
        }

        // Rotation
        if input.rotate_left  { self.angle -= ROTATION_SPEED * dt; }
        if input.rotate_right { self.angle += ROTATION_SPEED * dt; }

        // Thrust
        self.is_thrusting = input.thrust;
        if self.is_thrusting {
            self.velocity.x += self.angle.cos() * THRUST_POWER * dt;
            self.velocity.y += self.angle.sin() * THRUST_POWER * dt;
        }

        // Drag and speed limit
        self.velocity *= DRAG;
        let speed = self.velocity.length();
        if speed > MAX_SPEED {
            self.velocity = self.velocity.normalize_or_zero() * MAX_SPEED;
        }

        // Move
        self.position += self.velocity * dt;
        wrap_position(&mut self.position);
    }

    pub fn draw(&self) {
        if self.is_dead {
            return;
        }

        // Blink when invincible
        if self.invincible_timer > 0.0 {
            let blink = (self.invincible_timer * 10.0) as i32 % 2 == 0;
            if blink {
                return; // Skip drawing this frame = blinking effect
            }
        }

        // Draw thrust flame first (behind the ship)
        if self.is_thrusting {
            draw_thrust_flame(self.position, self.angle);
        }

        // Draw the ship body
        draw_ship_body(self.position, self.angle);
    }
}

fn draw_ship_body(position: Vec2, angle: f32) {
    let nose = Vec2::new(
        position.x + angle.cos() * 20.0,
        position.y + angle.sin() * 20.0,
    );
    let left = Vec2::new(
        position.x + (angle - 2.618).cos() * 12.0,
        position.y + (angle - 2.618).sin() * 12.0,
    );
    let right = Vec2::new(
        position.x + (angle + 2.618).cos() * 12.0,
        position.y + (angle + 2.618).sin() * 12.0,
    );
    draw_triangle(nose, left, right, WHITE);
}

fn draw_thrust_flame(position: Vec2, angle: f32) {
    let back = angle + std::f32::consts::PI;
    let flame_len = rand::gen_range(8.0, 20.0);
    let flame_w = rand::gen_range(3.0, 7.0);

    let base = Vec2::new(
        position.x + back.cos() * 8.0,
        position.y + back.sin() * 8.0,
    );
    let tip = Vec2::new(
        base.x + back.cos() * flame_len,
        base.y + back.sin() * flame_len,
    );
    let l = Vec2::new(
        base.x + (back + 1.5).cos() * flame_w,
        base.y + (back + 1.5).sin() * flame_w,
    );
    let r = Vec2::new(
        base.x + (back - 1.5).cos() * flame_w,
        base.y + (back - 1.5).sin() * flame_w,
    );

    draw_triangle(tip, l, r, ORANGE);
    draw_triangle(tip, base, r, RED);
}

fn wrap_position(pos: &mut Vec2) {
    let w = screen_width();
    let h = screen_height();
    if pos.x < 0.0 { pos.x += w; }
    if pos.x > w    { pos.x -= w; }
    if pos.y < 0.0 { pos.y += h; }
    if pos.y > h    { pos.y -= h; }
}

pub fn get_input() -> ShipInput {
    ShipInput {
        rotate_left: is_key_down(KeyCode::Left),
        rotate_right: is_key_down(KeyCode::Right),
        thrust: is_key_down(KeyCode::Up),
        shoot: is_key_pressed(KeyCode::Space),
    }
}

🧠 How the Blink Works

That one-liner packs a lot. Let's unpack it piece by piece:

let blink = (self.invincible_timer * 10.0) as i32 % 2 == 0;
  1. invincible_timer counts down from 2.0 (set by respawn logic in Chapter 9) to 0.0 over two seconds.
  2. * 10.0 scales it to 20.0 → 0.0, so the number changes faster.
  3. as i32 chops off the decimal. As the timer drops, this integer steps down: 20, 19, 18, 17...
  4. % 2 gives the remainder when divided by 2. Even numbers → 0. Odd numbers → 1.
  5. == 0 is true for even numbers, false for odd ones.

So blink toggles: true, false, true, false... about 10 times per second. When it's true, we skip drawing the ship. The player sees the ship flicker in and out—the universal game language for "you're temporarily invincible."

Change the 10.0 to 20.0 for a faster blink, or 5.0 for a slower one. The % 2 creates the on/off pattern; the multiplier controls the speed.

Your New main.rs

With the ship logic moved to its own module, main.rs becomes beautifully simple:

use macroquad::prelude::*;

mod ship;
use ship::Ship;

#[macroquad::main("Asteroids")]
async fn main() {
    let mut ship = Ship::new();

    loop {
        let dt = get_frame_time();
        clear_background(BLACK);

        // Gather input
        let input = ship::get_input();

        // Update
        ship.update(&input, dt);

        // Draw
        ship.draw();

        next_frame().await;
    }
}

Twenty lines. That's the power of organizing code into modules. The game loop is readable at a glance, and all the ship complexity lives in ship.rs where it belongs.

Making It Feel Good

The difference between a demo and a game is in the details. Here are small tweaks that dramatically improve how the ship feels:

Progressive Thrust

Instead of instant full thrust, ramp it up over a few frames. This gives a sense of weight and power buildup.

Rotation Inertia

The ship doesn't stop rotating instantly when you release the key. Add a rotational velocity that decays with drag, just like linear velocity. Feels much more spaceship-like.

Screen Shake on Death

When the ship dies, offset the camera randomly for half a second. It's a single line of code that sells the impact. We'll add this in Chapter 10.

Engine Sound (Audio)

A low hum when thrusting, pitch-shifted by velocity. Audio is optional but transformative. macroquad has basic audio support if you want to explore it later.

Key Takeaways

Organize Early: Ship logic in ship.rs

A clean main loop means you can add asteroids, bullets, and UI without chaos.

Input Structs: Collect all input at once

Decouples "what keys are pressed" from "what does the ship do about it."

Speed Cap: Always limit player velocity

Unbounded speed makes games uncontrollable. Pick a cap that feels fast but fair.

Flame = Angle + PI: Opposite direction every time

The thrust goes behind the ship. Simple addition, powerful visual.
